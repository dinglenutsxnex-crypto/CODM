# HammerScale

Android VPN-based traffic interceptor and packet editor for Shadow Fight 3 (`com.nekki.shadowfight3`).

HammerScale runs a local `VpnService` that routes the game's traffic through the device, parses its TCP/UDP/DNS packets, and lets you inspect and modify game protocol messages in real time through a floating overlay.

## How it works

- `TrafficVpnService` opens a TUN interface and pulls raw IP packets out of it.
- `PacketParser` decodes IPv4 headers plus TCP/UDP segments; `DnsPacket` decodes DNS queries/answers for logging.
- `TcpHandler` / `UdpHandler` reassemble per-connection streams and forward them to the real destination over sockets that bypass the VPN (`protect()`), so the game's own traffic isn't looped back into the tunnel.
- `GameProtocolParser` reads the reassembled application-layer bytes and turns them into typed `GameEvent`s (battle start, round result, win confirmation, etc.).
- `PacketInjector` builds and injects modified/forged packets back into the tunnel — this is how battle outcomes get patched server-side before the client sees them.
- `OverlayService` draws the floating control panel (dev mode + a simplified user mode) that arms/disarms intercepts and shows live connection and event data.
- `ConnectionViewModel` holds shared app state (active connections, current battle, live events) so the VPN service and UI stay in sync.

## Battle data pipeline

Some intercepts need to know a battle's `RoundsToWin` ahead of time. That data comes from `data/battles.json`, which is:

1. Generated by `scripts/extract_battles.py`, which fetches and decrypts the game's own config archive and extracts battle definitions.
2. Re-encrypted at build time (see `.github/workflows/build.yml`) into `app/src/main/res/raw/battles.enc`, which ships inside the APK and is decrypted on-device by `BattleConfig`.
3. Refreshed daily by `.github/workflows/update_battles.yml`, which reruns the extraction script and commits any changes.

## Project layout

```
app/src/main/java/com/nexora/hammerscale/
├── MainActivity.kt            entry point, permission + VPN start flow
├── TrafficVpnService.kt       VpnService: TUN setup, packet pump, intercept state
├── OverlayService.kt          floating overlay UI + control panel
├── SecurityModule.kt          root/tamper/debugger checks
├── BattleConfig.kt            loads and decrypts data/battles.json at runtime
├── AppState.kt                shared ConnectionViewModel singleton
├── GifView.kt, GlassCircleView.kt   custom overlay UI views
├── LogDownloader.kt           exports captured session logs
├── model/                     ConnectionEntry, GameEvent, ConnectionViewModel
├── net/                       PacketParser, DnsPacket, TcpHandler, UdpHandler,
│                               GameProtocolParser, PacketInjector
└── ui/                        RecyclerView adapters, socket detail screen

scripts/extract_battles.py     battle data extraction (see above)
data/battles.json              generated battle data, checked in for reproducibility
```

## Building

Requires JDK 17 and the Android SDK.

```
./gradlew assembleDebug
```

CI (`.github/workflows/build.yml`) builds a release APK on every push and uploads it as a workflow artifact.

This project modifies live network traffic for a third-party game client. It's intended for protocol research and personal use on your own accounts. You're responsible for how you use it and for complying with the target game's terms of service.
